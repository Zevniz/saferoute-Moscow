# Анализ качества маршрутизации SafeRoute

## Проблема

При построении маршрута возле финиша наблюдается "крюк" - маршрут не приходит точно в запрошенную точку.

## Результаты расследования

### 1. Snapping Distance

**Тестовый маршрут:**
- От: Кремль (55.7558, 37.6173)
- До: Третьяковская галерея (55.7496724, 37.6210752)

**Фактические точки маршрута:**
- Старт маршрута: (55.75598, 37.61748) - **22.5м** от запрошенной точки ✅
- Финиш маршрута: (55.75054, 37.62064) - **100.3м** от запрошенной точки ⚠️

### 2. Причина проблемы

**Valhalla автоматический snapping:**
- SafeRoute использует Valhalla для построения маршрутов
- Valhalla автоматически привязывает (snap) запрошенные координаты к ближайшему узлу дорожного графа
- Для пешеходного режима Valhalla ищет ближайший пешеходный сегмент
- В районе финиша ближайший пешеходный узел находится на расстоянии 100м

**Почему так происходит:**
1. В OSM данных может не быть пешеходного графа точно в запрошенной точке
2. Ближайший доступный пешеходный сегмент находится дальше
3. Valhalla корректно строит маршрут до ближайшей доступной точки
4. Это НЕ баг - это корректное поведение при отсутствии пешеходной инфраструктуры

### 3. Геометрия маршрута

- Точек в маршруте: 84
- Петель не обнаружено ✅
- Дубликатов координат нет ✅
- Порядок координат: [lon, lat] - корректный для GeoJSON ✅

### 4. Последние точки маршрута

```
79: (55.750647, 37.619913)
80: (55.750543, 37.620102)
81: (55.750511, 37.620161)
82: (55.750419, 37.620329)

```

Маршрут плавно подходит к финишу, нет резких поворотов.

## Выводы

1. ✅ **Маршрут построен корректно** - нет багов в коде
2. ⚠️ **Snapping на 100м** - это особенность пешеходного графа в данной локации
3. ✅ **Геометрия правильная** - нет петель, дубликатов, неправильного порядка координат
4. ⚠️ **Проблема в UX** - пользователь не понимает почему маршрут не приходит точно в точку
